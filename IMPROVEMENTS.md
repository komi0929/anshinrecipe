## 2026-01-09 - LINEログイン認証エラーの根本解決（自動修復・全件検索実装）

**ステータス**: 🚀 本番反映済み

**問題点**:
- **50名以上のユーザー環境での不具合**: `listUsers` APIが最初の50人しか返さないため、51人目以降の既存ユーザーが「存在しない」と誤認されログインがブロックされていた。
- **アカウント情報の不一致**: `@line.user` と `@line.anshin-recipe.app` の異なる形式のダミーメールが混在し、既存アカウントの特定に失敗していた。
- **データ不整合**: プロフィールはあるがAuthユーザーがいない、等の不整合状態になると復旧不能になっていた。

**変更内容**:
- **「自動復旧型」認証フローの実装**: 
  - プロフィール・Authユーザーの両面から検索を行い、欠損しているデータを自動的に再生成・紐付け。
  - ページネーションに対応した全ユーザー検索ロジック（`findAllUsersByEmails`）を導入し、ユーザー数増加に対応。
  - メールドメインの揺らぎを吸収し、既存アカウントを確実に特定。
- **コールバック画面の改善**: 
  - エラー発生時の表示時間を3秒から8秒に延長し、詳細なエラー原因を画面上で確認可能に。
  - エラー時の視認性を高める専用のUIスタイルを導入。

**関連ファイル**:
- `app/api/auth/line/route.js`
- `app/auth/callback/line/page.js`

**確認ポイント**:
- [x] 既存ユーザー（50名以降も含む）が正常にログインできること
- [x] 未ログイン（新規）ユーザーが正常に登録・ログインできること
- [x] エラー発生時に分かりやすいメッセージが表示されること
