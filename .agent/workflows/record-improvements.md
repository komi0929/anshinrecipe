# 改善記録ルール

**このルールは恒久的に適用されます。**

## 記録タイミング

以下のタイミングで必ず `IMPROVEMENTS.md` に記録を追加すること：

1. ユーザーが指示した改善・機能を実装完了した時
2. バグ修正を完了した時
3. UI/UX の改善を完了した時
4. 本番環境へのデプロイを完了した時

## 記録ファイル

- **ファイルパス**: `c:\Users\soyst\OneDrive\デスクトップ\あんしんレシピ\IMPROVEMENTS.md`

## 記録フォーマット

各改善は以下の形式で記録すること：

```markdown
## [日付] - [改善タイトル]

**ステータス**: ✅ 実装完了 / 🚀 本番反映済み / ⏳ 開発中

**変更内容**:
- 具体的な変更点1
- 具体的な変更点2

**関連ファイル**:
- `path/to/file1.js`
- `path/to/file2.css`

**確認ポイント**:
- [ ] 確認項目1
- [ ] 確認項目2
```

## 注意事項

- 日付は `YYYY-MM-DD` 形式で記載
- 新しい改善は**ファイルの上部**に追加（最新が上）
- ユーザーが「改善ログを見せて」と言ったら、このファイルの内容を表示する
- 目視確認が完了したら、確認ポイントのチェックボックスを更新する
